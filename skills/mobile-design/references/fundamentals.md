## Visual hierarchy

- Identify the one thing the user should notice first on each screen.
- Use size, weight, placement, and spacing before using extra color.
- Reduce competing emphasis. If three elements feel primary, none of them do.
- Keep supporting metadata quieter than the main content and action.

## Color and contrast

- Use color to signal meaning, grouping, and status, not just decoration.
- Reserve saturated accent colors for primary actions and important states.
- Do not rely on color alone for warnings, success, or selection states.
- Check contrast for text, icons, dividers, and disabled-looking controls that still need to be legible.
- For dark mode, reduce glare and preserve contrast without making every surface pure black.

## Typography

- Build a compact type scale with clear roles: display, title, body, caption, and labels.
- Prefer fewer text styles used consistently over many near-duplicates.
- Support dynamic type or larger text gracefully by allowing wrapping and flexible vertical space.
- Keep line lengths short on mobile and avoid dense paragraph blocks when the task is action-oriented.

## Layout and spacing

- Use spacing to show grouping and relationship. Related items should sit closer together than unrelated items.
- Keep margins and gutters consistent so screens feel intentional.
- Design for thumb reach and vertical scanning; important actions should not hide below unnecessary chrome.
- Use clear empty, loading, error, and success states so the layout does not collapse under real conditions.

## Accessibility review checklist

- Check whether text remains readable at larger font sizes.
- Check whether all tap targets feel comfortable for thumbs.
- Check whether focus, selection, pressed, disabled, and error states are visually distinct.
- Check whether screen-reader labels and reading order will still make sense from the structure.
- Check whether animations can be reduced or simplified for motion-sensitive users.
